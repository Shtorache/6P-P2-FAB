Desenvolva uma aplicacao frontend para consumir a API do projeto e integrar todas as funcionalidades do backend com uma interface web moderna, organizada e responsiva.

## Objetivo

Criar uma interface em React com Tailwind CSS que permita ao usuario interagir com o sistema completo, incluindo autenticacao, navegacao entre paginas, consumo de rotas protegidas e operacoes de CRUD dos recursos disponiveis no backend.

## Tecnologias obrigatorias

- React.
- Tailwind CSS.
- Docker.
- Docker Compose.

## Requisitos obrigatorios

- O frontend deve ser desenvolvido com React.
- A estilizacao deve ser feita com Tailwind CSS.
- O frontend deve consumir o backend ja desenvolvido na atividade anterior a prova.
- A aplicacao deve permitir integracao completa entre frontend e backend.
- O frontend deve rodar em container Docker.
- A execucao do projeto deve acontecer com Docker e Docker Compose.
- O projeto nao deve depender de execucao manual com `npm start`, `npm run dev` ou comandos equivalentes como fluxo principal de entrega.

## Funcionalidades minimas

- Criar uma tela de login integrada com a autenticacao do backend.
- Armazenar e utilizar o token JWT para acesso as rotas protegidas.
- Implementar controle de acesso para usuarios autenticados.
- Criar navegacao entre as telas da aplicacao.
- Criar interface para listar, cadastrar, editar e remover usuarios, conforme as permissoes definidas no backend.
- Criar interface para listar, cadastrar, editar e remover itens.
- Exibir mensagens de sucesso e erro para as operacoes realizadas.
- Garantir que a interface seja responsiva para desktop e mobile.

## Integracao entre frontend e backend

- Configurar a comunicacao com a API utilizando variaveis de ambiente.
- Garantir que as requisicoes autenticadas enviem o token JWT corretamente.
- Tratar respostas de erro da API, como falhas de login, acesso negado, validacoes e recursos inexistentes.
- Organizar a aplicacao de forma que o frontend reflita corretamente as regras de autenticacao e autorizacao implementadas no backend.
- Validar que o frontend consiga funcionar em conjunto com o backend por meio do Docker Compose.

## Requisitos de Docker

- Criar um `Dockerfile` para o frontend.
- Integrar frontend e backend em um mesmo `docker-compose.yml` ou em uma composicao equivalente da entrega.
- Garantir que os containers consigam se comunicar corretamente.
- Documentar a porta utilizada pelo frontend e a forma como ele acessa a API.

## Entregaveis

- Codigo-fonte do frontend.
- Estrutura em React organizada por componentes, paginas e servicos.
- Estilizacao com Tailwind CSS.
- Integracao funcional com o backend.
- Arquivo `Dockerfile` do frontend.
- Configuracao de `docker-compose.yml` contemplando frontend e backend.
- Arquivo `.env` ou `.env.example` com as configuracoes necessarias.
- Documentacao curta explicando como subir a aplicacao completa com Docker.
- No github

## Criterios de avaliacao

- Correta utilizacao de React e Tailwind CSS.
- Qualidade da integracao com o backend.
- Funcionamento das rotas protegidas no frontend.
- Organizacao do codigo e da interface.
- Responsividade.
- Funcionamento da aplicacao via Docker.
- Clareza da documentacao de execucao.
